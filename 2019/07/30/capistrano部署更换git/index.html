capistrano 部署中需要更换git 地址

  • 有个需求是需要在已有的部署文件中更换一个新的git源,并且切换分支
  • 因为现有部署项目已经在线上跑起来了，只是需要更换git源就可以了
  • 但是报出了以下错误
  • 后来查看发现是因为以前的git源存在于项目的/repo文件目录之下，所以需要删除/repo 文件，然后再次进行cap 操作就可以了
